# Contributing

This list is intended to become a reliable Awesome AI2-THOR index, so entries should be easy to verify.

## Entry Criteria

- The work must use, extend, benchmark, wrap, visualize, or generate data for AI2-THOR or a close derivative such as iTHOR, RoboTHOR, ProcTHOR, ManipulaTHOR, or ArchitecTHOR.
- Prefer primary links in this order: proceedings page, arXiv, official project page, official GitHub repository.
- Blog posts and tutorials are welcome only under tutorials/resources.
- If an item has code but no paper, put it under tools, datasets, or to-verify rather than papers.

## Format

Use this format for papers:

```markdown
- **[Paper Title](paper-url)** - Venue/status, Year.  
  One concise sentence about the AI2-THOR usage. Code: [owner/repo](repo-url).
```

Use this format for tools and datasets:

```markdown
- **[Project Name](repo-url)**  
  One concise sentence about what it adds to the AI2-THOR ecosystem.
```

## Verification Labels

- `conference/journal`: published or accepted in a named venue.
- `arXiv`: available as a preprint; include arXiv link.
- `project`: official or community code/project page without confirmed paper.
- `tutorial`: blog, course note, or walkthrough.
- `to-verify`: promising candidate that needs source cleanup.

## Maintenance Notes

- Keep entries sorted newest first within papers.
- Do not add long abstracts.
- Avoid claims like "SOTA" unless the source explicitly supports them and the date is clear.
- When a project page mentions AI2-THOR indirectly, add a short note explaining the connection.
